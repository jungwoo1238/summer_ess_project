## 0. 연구 목적과 산출물

**연구 질문:** "이 배전망에 ESS를 설치하는 것이 경제성이 있는가? 임계조건은 무엇인가?"
(→ "어디에 얼마나"에서 "경제성 갖는가"로 이동. 이연 위치무차별·손실~1%·손익분기 아슬아슬이 근거.)

**최종 의사결정 산출물: (b, S, E)** — 위치·전력용량·에너지용량.
- CAPEX가 (S, E)의 함수이며, 이 셋이 실제로 돈을 들여 결정하는 물리적 설비 사양.
- **q_ratio, P[t], Q[t] 스케줄은 최종 산출물이 아니라 "편익 평가의 산물"**이다.
  PCS는 정격 S 안에서 P·Q를 실시간 조절 가능하므로 q_ratio는 설비가 아니라 운영 파라미터.
  q*는 "이런 P/Q 배분이 편익을 최대화한다"는 운영 특성의 발견(설치 사양 아님).

**편익은 완전예지(perfect foresight) 상한이다.** 과거 데이터로 24시간을 미리 다 알고 최적화.
실제 운영자는 예측값(불완전)으로 운영하므로 실제 편익은 이 상한보다 다소 낮을 수 있다.
논문에는 "완전예지 상한"임을 명시만 하고 예측오차는 별도 모델링하지 않는다.
사이징 판단에는 상한으로 충분. Phase 1·Phase 2를 **동일한 완전예지 기준**으로 평가하므로
두 Phase의 비교는 공정(예측오차 편의가 양쪽에 동일하게 작용).

**Phase 구분:**
- Phase 1 (본 명세): 손실 + 이연 + 조달비, 유효+무효전력. "현재조건 경제성·임계조건".
- Phase 2 (나중): 태양광 추가(침투율 스윕). 시나리오 5→10개(맑음/흐림 분기).
  데이터 미확보, Phase 1 완료 후 착수. **코드는 시나리오 리스트 길이에 무관하게 설계**할 것.

## 2. 방법론 — 이중수준 최적화 (Nested)

### 상위 (PSO): 배치·용량·무효전력비 탐색
- **이질적 다수기.** n기 = 4n 차원: 각 기 `(b_i, S_i, E_i, q_ratio_i)` 독립.
- 1기부터 하나씩 늘리며 **순편익이 꺾이는 지점**을 탐색(최적 기수 자체가 결과).
  기수별 독립 최적화. 배전망 특성상 최적 기수는 작을 것으로 예상.
- **중복 배치 허용 (a)방식:** 같은 버스에 놓이면 sgen을 2개 별도 생성(병합 안 함).
  중복은 대개 좋은 해가 아니라 자연 도태됨. 특별 처리 불필요.
- 경계: `b∈[1,32] 정수`(슬랙 0 제외, 연속값 반올림), `S∈[0, 2.4] MVA`,
  `E∈[0, 10.2] MWh`, `q_ratio∈[0, 1]`.
  **뼈대(skeleton)에서는 q_ratio 경계를 [0,0]으로 고정**(Q=0). 원 제약은 구조적으로 유지.
- 상위는 `(b,S,E,q_ratio)`만 던지고 fitness 스칼라 하나를 받음. 편익 계산은 전부 하위+조류계산.

### 하위 (분리 LP, 시나리오별 독립)
> **결정: 통합 LP가 아니라 분리 LP.** 에너지는 AVG_DAYS만, 이연은 PEAK_DAYS만 계산하므로
> 한 시나리오 안에서 두 목적이 충돌하지 않는다. 운영변수 P_s[t]가 시나리오 간 독립이라
> 분리 최적화 = 통합 최적화(수학적으로 동일). 분리가 더 단순·디버깅 쉬움·확장 용이.
> (주의: AVG에서도 이연을 보거나 PEAK에서도 조달비를 계산하도록 바뀌면 통합/가중결합 필요.
>  현재 확정(AVG=에너지, PEAK=이연)을 유지하는 한 분리로 충분.)

- **AVG_DAYS 3개:** `min Σ_t SMP[t]·(P_ch[t] − P_dis[t])` (조달비=슬랙 유입 최소화).
- **PEAK_DAYS 2개:** 각자 자기 피크 `pk_s` 최소화. 최종 이연은 사후에 `max(pk_summer, pk_winter)`.
- **충·방전 변수:** 물리적으로 동시 충방전 불가 → 실제 운영변수는 부호 있는 단일 P[t].
  단 편익식·SOC 갱신의 효율 비대칭(충전 η_c 곱, 방전 η_d 나눔)을 표현하기 위해 개념상 P_ch/P_dis로 분리.
  LP 구현은 `P_ch, P_dis ≥ 0`(cvxpy nonneg) 두 비음변수로 두고, 상보성(동시 양수 배제)은
  **효율 손실(η<1)이 있어 최적성이 자동 보장**(동시 충방전은 항상 손해라 LP가 하나를 0으로).
- **손실은 LP가 모른다**(LP는 순부하 유효전력만 봄). 손실은 비선형(전류²)이라 LP에 못 넣음.
  LP는 조달비 대리로 스케줄을 짜고, **실제 손실 편익은 그 스케줄을 조류계산에 넣어 사후 측정**(결과 추종).
  근거: 손실 편익이 전체의 ~1%(왕복효율 손실보다도 작음)라 명시적 최적화 이득이 미미.
- 시간해상도 Δt = 1시간(24스텝).
- 무효전력: `Q[t] = q_ratio × sqrt(S_rated² − P[t]²)` (남는 PCS용량을 Q에).
  PCS 제약은 **원(circle)**으로: `sqrt(P² + Q²) ≤ S_rated`. |P|≤S로 축약 금지.
  Q는 하위LP가 결정 불가(순부하만 봄) → 상위변수.
- 솔버: cvxpy(로컬 환경에 설치 완료).

### SOC 제약
- `SOC[0] = SOC[24] = 50%` 고정. `SOC ∈ [5%, 95%]`(DoD 90%).
- **잔차상환·mean-shift 사영 전면 폐기.** LP가 SOC를 연속 결정변수로 두고
  등식제약(SOC[24]=SOC[0])·부등식제약(5~95%)으로 자연 처리하므로 사영 불필요.
- SOC 50% 근거: 대표일은 "특정 날"이 아닌 통계적 평균일이라, 시작 SOC를 특정할 근거가
  없으므로 가용범위(5~95%) 중앙 50%가 최대엔트로피(무정보) 선택. 계절 간 SOC 이월 없음과도 정합.
- **자기방전: SOC 갱신식에 시간별 반영 (복리 정확값).**
  `δ_hourly = 1 − (1 − 0.00067)^(1/24) = 2.7925633002e-05` (≈0.002793%/h).
  (선형근사 0.00067/24=2.7917e-5가 아닌 복리 환산값. 검산: (1−δ_hourly)^24 = 1−0.00067 = 0.99933.)
  `SOC[t+1] = SOC[t]·(1−δ_hourly) + η_c·P_ch[t]·Δt − P_dis[t]/η_d·Δt`.
  단기저장(하루 미만, SOC 매일 50% 리셋·날짜 간 이월 없음)이라 24h 누적 0.067%로 **영향 미미**
  (왕복효율 손실의 ~1/150). 완결성·질문차단 위해 넣되 결과엔 무영향. 무시해도 결론 불변.
  코드 상수: `SELF_DISCHARGE_HOURLY = 1 - (1 - 0.00067)**(1/24)` (매직넘버 대신 식으로).

### 왜 nested인가 (교대반복 아님) — 지형 왜곡이 없는 이유
원문제는 설비 x=(b,S,E,q)와 운영 y=(P,SOC)를 **동시 최적화**: `max_{x,y} f(x,y)`.
**항등식** `max_{x,y} f = max_x[ max_y f(x,y) ]` (항상 참). 안쪽 `g(x)=max_y f(x,y)`는
"설비 x 고정 시 최선 운영의 편익". nested는 상위 PSO가 x를 탐색하며 각 x의 fitness로 g(x)를 씀.

- **왜 왜곡 없나:** 상위가 x를 던질 때마다 하위 LP가 **그 x에 맞는 최적 운영을 즉석에서 정확히
  다시 풀어** g(x)를 반환. LP는 볼록→전역최적 보장. 따라서 상위가 보는 지형 g(x)는 **참된 지형**.
- **대조(교대반복):** 운영 y를 고정한 채 상위 x를 여러 번 평가하면, x가 바뀌어도 낡은 y로 평가되어
  fitness가 g(x)(참값)가 아닌 f(x,y_old)(열등값)가 됨 → 지형 왜곡, 수렴 실패, 평가 5~20배 증가.
- **가능한 이유:** (1) 하위가 볼록 LP라 전역최적 보장, (2) 하위 비용이 조류계산의 0.4%(~5ms)뿐이라
  매 평가마다 다시 풀어도 부담 없음. 둘 중 하나라도 깨지면 보장 상실 — 우리는 둘 다 만족.
- **정직한 단서:** 하위는 조달비 대리(슬랙 유입 min)로 y를 정하고 손실은 사후 측정(y 최적화 대상 아님).
  즉 하위 목적(조달비)과 최종 fitness(조달비+손실+이연) 사이 미세한 틈 존재. 손실~1%·조달비와 동방향이라
  실용적으로 무시 가능. 정확히는 "하위가 지배적 편익(조달비·이연)에 정확히 최적이라 지형 왜곡 실질 없음".

## 3. 목적함수

```
maximize J_net = B_energy + B_defer − Cost    [원/년]
fitness = −J_net + λ_V·Σ전압위반 + λ_line·Σ선로위반    (ALL_DAYS 검증)
```

### B_energy (손실 + 조달비 통합, SMP 기반) — AVG_DAYS만
```
B_energy = Σ_{s∈AVG} N_WD[s] × Σ_t [P_slack^base[s,t] − P_slack^ESS[s,t]] × SMP[s][t] × Δt
```
- 슬랙 유효전력 차이 × 시간별 SMP → **손실 편익 + 조달비 자동 합산(이중계상 없음)**.
- 항등식 `P_slack = Σ부하 + Loss − P_ESS` 에 의해 손실 감소분과 ESS 주입분이 모두 포착됨.
- **AVG_DAYS 3개만.** 최대일은 이연·제약 전용(최대일 데이터는 평균일 통계에 이미 녹아 있음 →
  최대일로 에너지를 또 세면 이중계상).

### B_defer (설비 이연, SMP 무관) — PEAK_DAYS, 단일 max
```
B_defer = c_cap × [max_{s∈PEAK,t} P_slack^base − max_{s∈PEAK,t} P_slack^ESS]
```
- **연중 최대 슬랙피크 1점**이 이연을 결정(여름최대일·겨울최대일 통합에서 단일 max).
- 일수가중 없음.

### Cost
```
CAPEX = 219,528.17 × E_rated[kWh] + 36,541.18 × S_rated[kVA]
OPEX  = 5,714.70 × E_rated[kWh] + 0.005564 × CAPEX
Cost  = CRF_20 × CAPEX + OPEX
```

### ★ 집계 방식이 항마다 다름 (이중계상 주의)
| 항 | 시나리오 | 가중 |
|---|---|---|
| 에너지 | AVG_DAYS | 일수가중 N_WD |
| 이연 | PEAK_DAYS | 슬랙피크 단일 max, 가중 없음 |
| 제약(페널티) | ALL_DAYS 전부 | — |

## 6. 손익분기 부등식 & E_rated 상한 재산정

### 손익분기 부등식 (조달비 포함, 근사식)
설비밀도(용량/전력 = E/S = x)당 이연+조달비 편익이 저장비용을 넘는 조건:
```
c_cap·η_peak + γ·DoD·x  >  (CRF+w)·(c_kWh·x + c_kW) + OM·x
```
- 좌변: 이연(c_cap·η_peak) + 조달비(γ·DoD·x). 우변: 연간화 CAPEX + O&M.
- η_peak=1.0(상한), γ = 왕복 순차익[원/유효kWh-yr] = Σ_s NWD[s]·(SMP_high·η − SMP_low/η).
등호에서 E/S 상한:
```
x_max = [c_cap·η_peak − (CRF+w)·c_kW] / [(CRF+w)·c_kWh + OM − γ·DoD]
```

### x_max 계산 (항별 수치, 1사이클/일·왕복효율 반영)
| 요소 | 값 |
|---|---|
| CRF+w | 0.076876 + 0.005564 = **0.08244** |
| 분자 = c_cap·1.0 − (CRF+w)·c_kW | 76,082.2 − 0.08244×36,541.18 = 76,082.2 − 3,012.45 = **73,069.75** |
| γ (조달비, AVG_DAYS·NWD가중) | summer 1,776.9 + winter 2,034.3 + shoulder 3,473.7 = **7,284.91** 원/유효kWh-yr |
| 분모 = (CRF+w)·c_kWh + OM − γ·DoD | 18,097.90 + 5,714.70 − 6,556.42 = **17,256.18** |
| **x_max** | 73,069.75 / 17,256.18 = **4.2344 kWh/kVA** |
| 손익분기 한계 지속시간 = x_max·DoD | 4.2344 × 0.90 = **3.811 h** |
> γ 각 시나리오: 순차익(원/유효kWh/일) = SMP_high·0.9487 − SMP_low/0.9487.
> summer 27.94(low104.35/high145.39), winter 33.46(128.40/177.93), shoulder 28.33(114.38/156.95).
> (참고) 조달비 제외 시: x_max=3.069, 지속시간 2.76h. 왕복효율 무시 시 x_max=5.2(과대).

### E_rated 상한 결정
- **E_rated 상한 = 10.2 MWh** (S=2.4MVA에서 x=4.25로 x_max 4.234를 살짝 상회 → 경계 잘림 방지).
- 실제 최적 E/S는 시그마 원식으로 PSO가 내부에서 탐색. 상한은 "최적점만 안 자르면 되는" 값.
- 손익분기 부등식 자체는 **근사식**이라 결론에서 비중 낮춤 — 정량 경제성 판정은
  후처리 지표(정확한 J_net·편익 분해)로. 부등식은 정성적 서술용(왜 구조적으로 아슬아슬한가).

### S_rated 상한 (별개 논리 — 충전 여유, DoD 무관)
- **S_rated 상한 = 2.4 MVA.** 상시운전용량(10,000 kVA)은 방향무관 선로한계이고, ESS 충전전력은
  그 시각 부하와 합산되어 초과 불가. 5개 시나리오 최저부하 시각 중 최소 충전여유(겨울 최대부하일
  최저부하 시각) = 2,473 kW → 2.4 MVA. **순시 전력 제약이라 DoD 무관**(DoD는 에너지축 E에만 반영).

## 8. 후처리 지표 (뼈대부터 포함, 최적화에 개입 안 함)

### 편익 분해 (목적함수는 통합, 후처리는 분리)
- **B_energy** (통합): 손실+조달비.
- **B_arb** (조달비 분리): `Σ_s N_WD[s] Σ_t (P_dis[t]−P_ch[t])·SMP[s][t]·Δt` — ESS 순방전×SMP.
- **B_loss** (손실 분리): `Σ_s N_WD[s] Σ_t (Loss^base−Loss^ESS)·SMP[s][t]·Δt`.
  **음수 가능**(LP가 조달비 좇아 심야 대량충전 시 그 시각 손실 증가). 오류 아님 — 주석 필수.
- **B_defer** (이연): 위 3절.
- **검산:** `B_arb + B_loss ≈ B_energy` (항등식. 안 맞으면 버그).

### 비용·순편익
- CAPEX, OPEX, Cost(=CRF_20·CAPEX+OPEX), **J_net**(=B_energy+B_defer−Cost).
- 각 편익 크기 비교 출력(예상: B_defer >> B_arb > B_loss).

### 운영·설비 지표 (프롬프트 6절)
1. **η_peak** = ΔP_peak / S_rated (상한 1.0).
2. **지속시간** = E_rated·DoD / S_rated (손익분기 한계 **3.81h** 참고, 조달비 포함 기준).
3. **E/S비** (kWh/kVA, x_max≈4.234 참고).
4. **EFC** (등가 전 사이클) = `Σ|P_net|·Δt / (2·E_usable)`, E_usable=E_rated·DoD.
   연간 = EFC×247. 두 가지 판정:
   - **활용도:** Lazard 350회/년과 비교(과소=활용저조 → 경제성 악화 원인, 과다=수명위협).
   - **20년 달력수명 위협:** 연간 EFC가 Lazard 수명가정(350회/년 = 7,000회/20년)을 넘는지.
     ※ 우리 연간 EFC는 **평일 247일 기준이라 과소평가**(주말·공휴일 제외). 실제 365일 운영은 더 많되,
       주말은 부하·SMP 낮아 운영강도가 평일보다 약해 **평일단순비례(365/247)만큼은 아님**.
     → **판정: 247일 기준(주값)이 이미 350회/년을 넉넉히 밑돌면 실제도 안전**(주말 EFC<평일 EFC라
       실제값이 평일비례보다 작음). 스케일링은 "주말=평일" 부정확 가정을 끌어들이므로 주값으로 안 씀.
     → **보조 상한점검:** 최악 가정(주말=평일)으로 365/247 스케일해도 350 미만인지 병기(과대경계).
5. **b* 분포** (30회 독립실행). 위치 무차별하면 흩어짐 → 그 자체가 결과.
6. **q*_ratio** (무효전력 최적값. q*≈0 예상 → "운영자가 Q 고려 안 해도 되는 이유"가 기여).

### 추가 지표
7. **평균일/최대일 스케줄 유사도.** 두 스케줄을 겹쳐 정량화.
   유사하면 "일관된 운영"의 방증, 다르면 "차익 유인 vs 피크저감 유인 충돌"의 발견.

   (평균일은 가격을, 최대일은 부하를 좇지만 둘이 같이 움직여 형상이 유사해질 수 있음).

## 9. 연구 방향성 (합의)

3층 결론구조:
1. **손익분기 부등식** (구조적 조건, 정성적 서술용): η상한 1.0 → 최대지속 **3.81h**(조달비 포함).
   `c_cap·η_peak + γ·DoD·(E/S) > (CRF+w)(c_kWh·E/S + c_kW) + OM·(E/S)` (6절 참조, 근사).
2. **PSO로 달성 가능한 최대 η 도출 → 경제성 판정** (정량, 후처리 지표로).
3. **Phase 2가 격차 메우는 조건** (PV 침투율, 조달비).

**우선순위: Phase 1 vs Phase 2 비교가 우선.** Phase 2에서 양수 결론을 지향하되,
Phase 1의 부호 자체를 목표로 삼지 않음. 두 Phase를 **동일 기준**으로 평가해 공정 비교하는 것이 핵심.
(※ 편익을 인위적으로 낮추거나 높이는 선택 금지 — 비교를 왜곡함.)

### 운영 모드 전환의 정합성 (완전예지 상한의 방어)
운영자는 평범한 날엔 조달비 편익 최대화(저가충전-고가방전), 연중 최대부하 예상일엔 피크저감 최대화.
이 모드 전환이 정당한 이유: **최대부하일 예측이 실제로 잘 맞음**(폭염·한파 예보·요일·계절로 며칠 전 예측).
피크일 판별은 예측 정확도가 높아 완전예지와 실제의 괴리가 작음. 최대일은 특정 하루가 아닌
**연중 상위 5일 평균**이라 "이런 극단일이 오면 이렇게 대응한다"는 대표적 피크대응(운영지침 아님).
같은 설비를 그날 조건에 따라 다른 정책으로 운영 — 설비는 하나, 운영정책만 전환.

### 배전망을 택한 근거
편익 귀속 주체 명확(단일 배전사업자), 배치(b) 변수가 의미를 갖는 유일한 층위,
R/X 高 배전망 고유 물리(유효전력↔전압·손실 직결, 무효전력 전압개선 미미),
분산전원 통합 이슈의 현장(Phase 2 정합), 정책적 시의성(11차 계획으로 배전 투자 확대),
IEEE 33-bus 표준 벤치마크.

### 평일만 사용한 근거 (층위별)
편익을 낮추려는 게 아니라 **평가 범위를 "전형적 운영일"로 정의**한 것.
1. **정의:** 평일은 산업·상업 정상가동일로 배전 계획·운영의 기준 조건. 주말·공휴일은 수요패턴이
   질적으로 다른 별개 모드 → 하나의 대표일로 평균 내면 두 모드가 뭉개진 혼합일(어느 쪽도 대표 못 함).
   즉 "대표성 포기"가 아니라 "이질적 모드를 안 섞으려는 분리". (필요시 주말 대표일 별도 확장 가능.)
2. **이연 물리 (견고):** 설비 이연은 연중 최대 피크가 결정 → 거의 평일 발생(산업수요). 주말은 피크 안 만듦.
   주말 포함 여부와 이연 편익 무관(인과 깨끗함).
3. **비교 일관성:** 주말 포함 시 일평균 편익↓·일수↑라 연간 총합 방향 불분명하나, 에너지 편익이 총편익에서
   작고(이연 지배) 이연은 평일 불변 → 결론(Phase 1 vs 2 비교) 안 바뀜. **Phase 1·2를 동일 평일기준으로
   평가하는 한 비교는 공정.**
4. (보조) 전력분석에서 평일/주말/공휴일 분리는 요일·공휴일 효과로 인한 표준 관행.
> 심사 대비: "주말 편익은?" → "평일 기준 편익을 평가했고 주말·공휴일은 수요·이연 기여가 작아 제외.
> 연간 총편익 완전산정엔 주말 대표일 추가가 후속 과제." (음수/양수 어느 쪽으로도 편향 없는 서술.)

## 부록 A. 모듈 구조 (제안)

폴더: `summer_ess_project/`. 분리 원칙 = **"바뀌는 이유가 같은 것끼리 묶고 다른 것끼리 분리"**
(파라미터 수정은 params.py만, LP 수정은 lower_lp.py만 건드리게).

```
main.py            # 진입점(오케스트레이터). 기수 루프(1기→2기→…) → PSO 실행(병렬) → 후처리 → 저장.
pso_core.py        # (검증완료) 범용 PSO 엔진. 벡터화 objective 인터페이스. ESS 문제 몰라도 됨. 손대지 않음.
build_net.py       # 계통 준비: case33bw 로드 + 22.9kV/10MVA 스케일 + 선로정격 주입. 워커가 시작 시 1회 호출.
lower_lp.py        # 하위 분리 LP. solve_avg(조달비 min), solve_peak(피크 min). 입력(부하곡선,S,E,q)→스케줄 P[t]. cvxpy.
evaluate.py        # 평가 지휘자. evaluate_particle(x): 하위LP → 24h 조류계산 → 결과수집 → benefits → fitness. PSO↔나머지 다리.
benefits.py        # 편익·비용 공식(순수함수): B_energy/B_arb/B_loss/B_defer/CAPEX/OPEX/Cost.
postprocess.py     # 후처리 지표(8절). 최적해 받아 계산·출력. 최적화 비개입 사후분석이라 격리.
params.py          # 모든 상수 창고: CRF·비용·c_cap·LOAD/SMP·경계·PSO초기값. 나머지 전부 여기서 가져옴. "숫자는 전부 여기".
test_lp.py         # LP 해석해 대조 검증(7절). 단순 SMP 장난감케이스로 lower_lp 정답 확인.
```

**데이터 흐름:** params(상수) → build_net(계통) → [main이 PSO 시작] → PSO가 x=(b,S,E,q) 제안 →
evaluate가 lower_lp(스케줄)+조류계산+benefits(편익)로 fitness 반환 → PSO가 x 갱신 반복 →
최적해 → postprocess(지표). test_lp는 곁가지로 lower_lp 검증.

**분리 이유:** (1) 관심사 분리(계통·LP·편익·최적화·후처리가 각각 다른 이유로 바뀜),
(2) 검증 격리(pso_core=검증완료, test_lp=검증도구), (3) Phase 2 확장(params 딕셔너리만 5→10 확장,
로직은 리스트 순회라 불변). Colab 실행은 `!python main.py` 단일 진입점(파일 수와 무관).
